# XPC3006: Custom API name must be a compile-time constant

## Severity

Warning

## Description

The type-safe `RegisterAPI<TService>(name, handlerMethodName)` overload names its generated
`{ApiName}Request`/`{ApiName}Response` classes and `ActionWrapper` after the API name. The source
generator therefore needs the name to be a **compile-time constant** (a `nameof(...)`, a `const`, or
a string literal).

If the name is a non-constant expression (a local variable, parameter, method call, etc.), the
generator cannot determine the class names, so **no `ActionWrapper` is generated**. The registration
then fails at runtime with an `InvalidPluginExecutionException` because no generated wrapper can be
discovered.

## ❌ Example of violation

```csharp
public class SomeCustomApi : Plugin
{
    public SomeCustomApi()
    {
        var name = ResolveName();   // not a compile-time constant

        // XPC3006: Custom API name must be a compile-time constant
        RegisterAPI<CallbackService>(name, nameof(CallbackService.Handle))
            .AddResponseProperty("StatusCode", CustomApiParameterType.Integer);
    }
}
```

## ✅ How to fix

Use a `nameof(...)`, a `const`, or a string literal:

```csharp
RegisterAPI<CallbackService>(nameof(SomeCustomApi), nameof(CallbackService.Handle))
    .AddResponseProperty("StatusCode", CustomApiParameterType.Integer);

// or
RegisterAPI<CallbackService>("contextand_SomeCustomApi", nameof(CallbackService.Handle))
    .AddResponseProperty("StatusCode", CustomApiParameterType.Integer);
```

## Note

This applies only to the typed `RegisterAPI<TService>(name, handlerMethodName)` overload. The
action-based overloads (`RegisterAPI<TService>(name, Action<TService>)` /
`RegisterCustomAPI(name, ...)`) do not generate code and accept any name expression.
